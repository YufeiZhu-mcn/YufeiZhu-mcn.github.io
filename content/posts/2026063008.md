### 第三步：编写调用代码（核心示例）

现在，我们编写一个简单的服务类 `Grok4ChatService.java`。核心操作就是新建一个OpenAI客户端，然后指定`baseUrl`为千聚的地址。

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageRole;
import java.time.Duration;
import java.util.ArrayList;
import java.util.List;

public class Grok4ChatService {

    public static void main(String[] args) {
        // 1. 从千聚平台获取的API Key
        String apiKey = "sk-your-qianju-api-key"; // 替换成你的真实Key

        // 2. 千聚兼容OpenAI的Base URL
        String baseUrl = "https://www.qianjuai.com/v1";

        // 3. 创建OpenAiService实例，设置超时（确保稳定）
        OpenAiService service = new OpenAiService(baseUrl, apiKey, Duration.ofSeconds(60));

        // 4. 构建请求消息
        List<ChatMessage> messages = new ArrayList<>();
        messages.add(new ChatMessage(ChatMessageRole.SYSTEM.value(), "你是一个专业的Java技术顾问。"));
        messages.add(new ChatMessage(ChatMessageRole.USER.value(), "请解释一下Java中虚拟线程（Virtual Threads）的优势。"));

        // 5. 指定Grok4模型名称
        ChatCompletionRequest chatCompletionRequest = ChatCompletionRequest.builder()
                .model("grok-4") // 这里是关键，指定模型为Grok4
                .messages(messages)
                .maxTokens(1024)
                .temperature(0.7)
                .build();

        try {
            // 6. 发起请求并获取响应
            System.out.println("正在调用Grok4...");
            service.createChatCompletion(chatCompletionRequest)
                    .getChoices()
                    .forEach(choice -> {
                        System.out.println("Grok4回复: " + choice.getMessage().getContent());
                    });
        } catch (Exception e) {
            System.err.println("调用Grok4时发生错误: " + e.getMessage());
            e.printStackTrace();
        } finally {
            // 7. 释放资源
            service.shutdownExecutor();
        }
    }
}


**关键点解读：**

-   **`new OpenAiService(baseUrl, apiKey, ...)`**：这行代码是魔法所在。我们仅仅将`baseUrl`从官方的 `https://api.openai.com/v1` 改为了千聚的 `https://www.qianjuai.com/v1`。API Key也从OpenAI的换成了千聚的。
-   **`model("grok-4")`**：在千聚平台上，Grok4的模型名称就是 `grok-4`。你不需要关心xAI后端的复杂命名。这行代码告诉千聚，你要调用的是Grok4模型。
-   **超时处理**：网络环境再稳定，也建议设置合理的超时时间。这里设置了60秒超时，足够处理绝大多数正常请求。如果网络极差，你会收到超时异常，但这比程序卡死要好得多。
-   **异常捕获**：在`catch`块中，可以记录详细的错误信息。如果请求失败，检查你的API Key是否正确，以及网络是否能直通千聚服务器。
