## 踩坑指南：这 5 个坑，提前避开

我整理了几个容易出问题的地方，你可以直接保存下来对照：

### 1. API 地址最后没加 `/v1`
就像我上面说的，`https://www.qianjuai.com` 不行，必须写成 `https://www.qianjuai.com/v1`。很多工具（如 Cursor、LobeChat）的配置框里，地址后面不会自动补 `/v1`，要自己手动补上。

### 2. 忘记把 key 换成千聚的
你原项目里的 OpenAI key 是 OpenAI 直接发的，千聚的 key 是你在后台生成的。两者不一样。第一次接入时，100% 会遇到这个问题。

### 3. 用错 model 名字
千聚支持的 model 名称是标准化过的，比如模型可能叫 `gpt-4o`、`claude-3-5-sonnet-20241022`、`gemini-2.5-pro-exp-03-25`，跟官方名字一致。但有些国产模型可能会多一些前缀（比如 `deepseek-chat`），建议先查一下千聚后台的模型列表，避免用错。

### 4. 超时设置太小
直连时，如果网络环境不好，默认的超时时间（比如 60 秒）可能够用。但换成千聚后，因为经过一次中转，流式响应时最好把超时设得长一点，比如 300 秒。在 OpenAI 库中通过 `timeout` 参数设置：

python

client = OpenAI(
    api_key="你的key",
    base_url="https://www.qianjuai.com/v1",
    timeout=300.0  # 留足时间
)

### 5. 忽略 Key 的权限
千聚后台可以生成多个 key，每个 key 可以设定不同的权限（比如只允许调用某些模型、限制用量）。如果你的 key 一开始调用失败，注意检查后台的 key 权限配置。
