## 先说说 Qwen-Turbo 的 Java 接入示例：到底有多简单

很多人以为调用大模型 API 必须用 Python，其实 Java 生态也完全能搞定。官方提供了 OpenAI 兼容的接口，只要你把 base_url 指向正确的地址，一套完整的 HTTP 请求就能跑通。

这里我拿一个最简单的 Java 示例来说明：

java
import okhttp3.*;
import org.json.JSONObject;
import java.io.IOException;

public class QwenTurboDemo {
    public static void main(String[] args) throws IOException {
        String apiKey = "your-api-key";
        String url = "https://www.qianjuai.com/v1/chat/completions";

        OkHttpClient client = new OkHttpClient();
        MediaType mediaType = MediaType.parse("application/json");
        JSONObject requestBody = new JSONObject();
        requestBody.put("model", "qwen-turbo");
        requestBody.put("messages", new JSONObject[]{new JSONObject(){{put("role", "user"); put("content", "你好，请简单介绍下你自己。");}}});

        Request request = new Request.Builder()
                .url(url)
                .post(RequestBody.create(requestBody.toString(), mediaType))
                .addHeader("Authorization", "Bearer " + apiKey)
                .build();

        try (Response response = client.newCall(request).execute()) {
            System.out.println(response.body().string());
        }
    }
}


代码写出来就这几行，逻辑一目了然：指定模型、传入消息、带上 API Key、发请求、拿到返回结果。如果你之前用过 OpenAI 的接口，那连换行都不用改，直接替换 base_url 就完事。

## 隐藏收费陷阱曝光：你以为的免费，其实到处是“坑”

为什么说坑无处不在？我扒了几个常见的收费陷阱，大家自己对照看看：

**陷阱一：按请求次数计费，而不是按 Token 计费**

官方文档上写着“按 Token 计费”，但很多第三方平台接入时，却悄悄改成了按请求次数计费。一次请求不管多短，都按固定价格收费。你的应用如果频繁发小请求，成本直接翻倍。

**陷阱二：不同模型“捆绑销售”，便宜模型反而贵**

有些平台为了让开发者用上 Qwen-Turbo，设置了“入门套餐”，里面搭配了其他模型一起卖。你实际想用便宜模型，但套餐价格包含了贵模型的分摊费用，最终付费比单独买贵得多。

**陷阱三：“首单免费”后，自动续费难以取消**

很多平台为了拉新，推出“首单免费”活动——比如新用户注册送 0.2 美元额度。但一旦用了这个额度，系统会自动开启订阅，账单期一到就开始扣费，而取消订阅的入口藏得非常深，要在三级菜单里找半天。

**陷阱四：计费单位模糊，小数点后直接进位**

有的平台计费单位是“次请求”，每次请求不足 1 元就按 1 元收费。这意味着你一次简单对话只用 0.003 元，系统也能收你 1 元。一次两次还小，要是跑个 1000 次请求呢？1000 元就这么没了。

这些陷阱在 Qwen-Turbo 的 Java 接入场景里特别常见。因为 Java 开发一般用于长期运行的后端服务，对 API 的调用量大，频繁。你如果没仔细看计费规则，很容易被“按需计费”这几个字误导，以为很便宜，结果账单下来直接傻眼。
