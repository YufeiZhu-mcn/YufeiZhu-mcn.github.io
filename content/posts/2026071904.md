### DeepSeek的Node.js示例接入——改一行代码的事

假设你已经在用`openai`的Node.js SDK，而且已经在用DeepSeek的API。现在想切换到千聚平台，统一管理所有模型的调用。

**步骤1：安装依赖（如果你用的是原生SDK）**

**步骤2：核心代码替换**

这是最令人舒适的地方：你不需要重写任何业务逻辑。你只需要在初始化客户端时，替换`baseURL`和`apiKey`。

// 老代码：调用DeepSeek官方接口

  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: 'https://api.deepseek.com/v1' // 旧地址
});

// 新代码：统一接入[千聚ai聚合平台](https://www.qianjuai.com/)

const client = new OpenAI({
  apiKey: process.env.QIANJUAI_API_KEY, // 在千聚申请的Key
  baseURL: 'https://www.qianjuai.com/v1' // 统一入口
});


**步骤3：完成调用**

之后的所有调用方法完全不变。你可以像这样调用DeepSeek：

javascript
async function callDeepSeek(prompt) {
  const response = await client.chat.completions.create({
    model: 'deepseek-chat', // 或 deepseek-reasoner
    messages: [{ role: 'user', content: prompt }]
  });
  return response.choices[0].message.content;
}


**关键逻辑解读**：
- 千聚的API完全兼容OpenAI格式，所以你熟悉的`chat.completions.create`、`stream`、`tool_calls`等所有参数全部适用。
- 替换`baseURL`后，你可以在同一个client对象上调用任意支持模型（如`gpt-4o`、`claude-sonnet-4-20250514`），无需额外初始化别的SDK。这是“统一接入”的精髓。
