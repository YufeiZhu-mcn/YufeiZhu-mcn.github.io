### 第二步：编写核心调用类

创建一个Java类，比如`QwenTurboClient.java`。把以下代码粘贴进去，你只需要替换`YOUR_API_KEY`为你在千聚后台获取的密钥即可。

java
import com.alibaba.fastjson2.JSON;
import com.alibaba.fastjson2.JSONObject;
import org.apache.hc.client5.http.classic.methods.HttpPost;
import org.apache.hc.client5.http.impl.classic.CloseableHttpClient;
import org.apache.hc.client5.http.impl.classic.HttpClients;
import org.apache.hc.core5.http.io.entity.StringEntity;
import org.apache.hc.core5.http.io.entity.EntityUtils;
import org.apache.hc.core5.http.ContentType;
import java.io.IOException;

public class QwenTurboClient {

    // [千聚api中转站](https://www.qianjuai.com/)的兼容接口地址
    private static final String BASE_URL = "https://www.qianjuai.com/v1";
    // 替换为你在千聚后台获取的API Key
    private static final String API_KEY = "sk-your-api-key-here";

    public static void main(String[] args) throws IOException {
        // 1. 构建请求体（符合OpenAI Chat Completion格式）
        JSONObject requestBody = new JSONObject();
        requestBody.put("model", "qwen-turbo");
        requestBody.put("messages", JSON.parseArray("[{\"role\":\"user\",\"content\":\"你好，请用一句话介绍[千聚ai中转站](https://www.qianjuai.com/)。\"}]"));
        requestBody.put("temperature", 0.7);
        requestBody.put("max_tokens", 200);

        // 2. 发起请求
        CloseableHttpClient httpClient = HttpClients.createDefault();
        HttpPost httpPost = new HttpPost(BASE_URL + "/chat/completions");
        httpPost.setHeader("Authorization", "Bearer " + API_KEY);
        httpPost.setHeader("Content-Type", "application/json");
        httpPost.setEntity(new StringEntity(requestBody.toJSONString(), ContentType.APPLICATION_JSON));

        try {
            // 3. 执行请求并解析响应
            String responseString = httpClient.execute(httpPost, response -> {
                return EntityUtils.toString(response.getEntity());
            });

            // 4. 提取AI的回复内容
            JSONObject responseJson = JSON.parseObject(responseString);
            String content = responseJson.getJSONArray("choices")
                                         .getJSONObject(0)
                                         .getJSONObject("message")
                                         .getString("content");
            System.out.println("AI回复: " + content);
        } finally {
            httpClient.close();
        }
    }
}


代码说明：
- **请求地址**：你访问的是`BASE_URL + "/chat/completions"`，这是标准的OpenAI Chat API路径，[千聚ai中转站](https://www.qianjuai.com/)完美兼容。不需要额外拼接奇怪的路径。

## 常见报错及避坑指南（图文解析）

即使代码写对了，新手也容易掉进几个常见的坑里。下面结合我实际遇到并解决的错误，给你一份图文避坑清单。

### 1. 401 Unauthorized （认证失败）

**错误现象**：返回HTTP 401状态码，响应体包含`{"error":{"message":"Incorrect API key provided","type":"invalid_request_error"}}`。

**根本原因**：API Key复制错误，或者Key已过期/被删除。

**解决方案**：
- 回到千聚后台，在“API Keys”页面，点击你Key旁边的“显示”按钮，重新复制。注意不要漏掉开头的`sk-`。
- 如果你重新生成了Key，旧Key会立即失效，请在代码中更新。
- 检查代码中是否有多余的空格或换行： `"Bearer " + API_KEY` 这一行，`Bearer`后面必须有一个空格。

### 2. 404 Not Found （路径错误）

**错误现象**：请求返回404，页面提示路径找不到。

**根本原因**：请求地址拼写错误，比如把`/chat/completions`写成了`/chat/completion`（少了个s）。

**解决方案**：
确保你的POST请求地址是完整的：`https://www.qianjuai.com/v1/chat/completions`。这是[千聚ai中转站](https://www.qianjuai.com/)完全兼容OpenAI的路径，不要自作主张地添加额外路由或改变路径。

> 避坑提示：某些旧教程会建议你用`/v1/completions`，这对应的是OpenAI旧版的文本补全接口，现已不推荐。[千聚ai中转站](https://www.qianjuai.com/)也支持`/completions`，但最好用新的`/chat/completions`避免后续模型兼容性问题。

### 3. 404 或 “Model not found” （模型错误）

**错误现象**：请求返回`{"error":{"message":"The model `qwen-turbo` does not exist","type":"invalid_request_error"}}`。

**根本原因**：你没有使用正确的模型名，或者该模型在当前分组下被禁用。

**解决方案**：
- 检查你的代码中`model`字段是否写为`qwen-turbo`，注意大小写。千聚平台大小写敏感，必须全部小写。
- 登录千聚后台，进入“模型”页面，搜索“qwen-turbo”，确认它位于你当前使用的分组（比如默认分组）中。如果你不小心把API Key绑定到了只支持OpenAI的“官转分组”，那自然会找不到Qwen模型。重新创建一个绑定在“默认”或“限时特价”分组的API Key即可。

### 4. 超时或连接超时 （网络问题）

**错误现象**：Java代码抛出`java.net.SocketTimeoutException: Read timed out`或`Connect timed out`。

**根本原因**：网络环境无法访问`www.qianjuai.com`，或者防火墙/代理拦截了HTTPS请求。

**解决方案**：
- 首先，在浏览器或命令行中使用`ping www.qianjuai.com`和`curl -X POST https://www.qianjuai.com/v1/chat/completions`测试。如果返回正常状态码，但Java代码超时，请检查你的Java进程是否设置了代理。很多国内开发环境会配置全局代理（如`http_proxy`），导致请求走错了出口。
- 在`HttpPost`之前添加如下配置，强制不走代理（注意：这取决于你的代码运行环境）：
java
System.setProperty("java.net.useSystemProxies", "false");

- 如果公司网络限制了对外HTTPS连接，请使用VPN或换到家庭网络测试。

## 进阶：集成到Spring Boot项目并处理流式输出

对于生产环境，你可能需要支持流式输出（Streaming），而不是等待全部结果返回。[千聚ai中转站](https://www.qianjuai.com/)完美支持SSE（Server-Sent Events）流式响应。这里提供一个Spring Boot + WebFlux的高效示例简版（确保你的项目引入了`spring-boot-starter-webflux`）。

java
import org.springframework.web.reactive.function.client.WebClient;
import reactor.core.publisher.Flux;

public class QwenStreamingExample {
    private static final String BASE_URL = "https://www.qianjuai.com/v1";
    private static final String API_KEY = "sk-your-key";

    public Flux<String> streamChat(String question) {
        WebClient client = WebClient.builder()
                .baseUrl(BASE_URL)
                .defaultHeader("Authorization", "Bearer " + API_KEY)
                .build();

        String requestBody = """
                {
                    "model": "qwen-turbo",
                    "messages": [{"role": "user", "content": "%s"}],
                    "stream": true
                }
                """.formatted(question.replace("\"", "\\\""));

        return client.post()
                .uri("/chat/completions")
                .bodyValue(requestBody)
                .retrieve()
                .bodyToFlux(String.class);
    }
}


注意：流式请求时，`"stream": true` 必须显式设置为`true`，[千聚ai中转站](https://www.qianjuai.com/)才会启用SSE模式。否则默认是非流式响应。
