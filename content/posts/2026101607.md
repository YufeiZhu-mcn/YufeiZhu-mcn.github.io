    api_key="sk-这里是你的APIkey",  # 替换为你刚才复制的Key

knowledge_base = {
    "退款政策": "我们支持7天无理由退换货，运费由我们承担。",
    "发货时间": "默认下单后48小时内发货，遇节假日顺延。",
    "会员权益": "会员享有免运费、专属折扣和生日礼物。"

    # 第一步：优先匹配本地知识库

            {"role": "user", "content": user_query}
        ],
        max_tokens=200,
        temperature=0.7
    )
    return response.choices[0].message.content

## 第三步：99%的人都会遇到的“坑”——常见错误代码解析

光写代码不行，还得能排错。我在跑通千聚API时，踩过下面这几个坑，你也大概率会遇到。我把它们列出来，附带解决方案。

### 错误1：`AuthenticationError: Incorrect API key provided`

-   **问题现象**：调用失败，提示API Key错误。
-   **原因**：你复制的API Key可能不完整，或者key前面带了一个不可见的空格字符。
-   **解决方案**：
    -   重新复制Key，确保没有遗漏。
    -   在代码里 `api_key="你的key"` 的引号后面，检查是否有多余空格。
    -   可以直接到千聚后台“API Key”页面，重新生成一个新的Key，并立刻替换。

### 错误2：`NotFoundError: The model 'gpt-4.5-turbo' does not exist`

-   **问题现象**：模型不存在或者不支持。
-   **原因**：你写死了某个模型名，但这个模型在千聚平台近期下架了，或者模型名拼写错误（比如把`gpt-4o`写成了`gpt-4`）。
-   **解决方案**：
    -   登录千聚后台，打开“模型列表”，查看当前支持的模型名称。
    -   在代码中，**永远不要硬编码模型名**。建议写一个动态获取的方法，或者将这个名称放到配置文件中。比如：

    python
    # 推荐做法：从数据源获取当前活跃的模型名
    model_name = "gpt-4o"  # 改为从后台获取的最新名称
    

### 错误3：`RateLimitError: Requests rate limit exceeded`

-   **问题现象**：请求被限流，提示超出速率限制。
-   **原因**：你的程序在极短时间内发送了大量请求。
-   **解决方案**：
    -   使用`sleep()`函数，在循环请求之间加一点延迟。例如：
    python
    import time
    for i in range(10):
        # 每2秒发送一次请求
        response = client.chat.completions.create(...)
        time.sleep(2)
    
    -   或者使用异步并发库（如`asyncio`），并控制最大并发数为5或10。

### 错误4：`BadRequestError: 400, 'messages' is not valid`

-   **问题现象**：报错信息说messages结构不合法。
-   **原因**：`messages`列表里必须包含`role`和`content`，并且roles必须是`system`、`user`或`assistant`三者之一，大小写敏感。
-   **解决方案**：检查你的`messages`参数。常见错误是写成了`"role": "User"`（首字母大写），应该是全小写`"role": "user"`。

    python
    # 正确的写法
    messages = [
        {"role": "system", "content": "你是一个助手。"},
        {"role": "user", "content": "你好"}
    ]
    

### 错误5：连接超时或 `ConnectionError`

-   **问题现象**：代码卡住很久后报错网络异常。
-   **原因**：你的网络环境可能不稳定，或者错误地设置了代理，导致请求未能正常到达`www.qianjuai.com`。
-   **解决方案**：
    -   **确保国内可直接访问**：千聚不需要梯子，如果你电脑上挂着代理，请在请求代码前关闭代理，或者在代码里移除代理设置。
    -   可以通过在代码开头加上`os.environ['NO_PROXY'] = 'www.qianjuai.com'` 来强制不走代理。
    -   检查DNS解析是否正常，可以用 `ping www.qianjuai.com` 测试连通性。

## 第四步：如何保证“100%成功率”？

所谓成功率，不仅仅是代码不报错，而是每一次调用都能返回有效的AI结果。要做到这一点，请遵循以下“黄金准则”：

1.  **错误重试机制**：不要只写一个`try...except`，而是要实现指数退避的重试。意思就是，当遇到5开头的服务器错误（如`500`、`502`、`503`）时，程序应该自动等待1秒、2秒、4秒再重试，直到成功或最大重试次数。

    python
    import time
    MAX_RETRIES = 3
    for attempt in range(MAX_RETRIES):
        try:
            response = client.chat.completions.create(...)
            break
        except Exception as e:
            if attempt == MAX_RETRIES - 1:
                raise
            time.sleep(2 ** attempt)
    

2.  **模型降级**：如果你的主模型（比如`gpt-4o`）暂时不可用，代码应该能自动切换到备选模型（比如`gpt-3.5-turbo`或`deepseek-v3`）。千聚提供了500+模型供你切换，这不浪费这个优势。

3.  **固定网络环境**：始终在“干净”的国内网络环境下运行程序。如果有条件，将后端代码部署在国内云服务器上，调用千聚的延迟更低，成功率更高。
