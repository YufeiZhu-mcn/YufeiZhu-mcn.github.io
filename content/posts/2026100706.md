这是最容易出错的地方。千万别把你的API key硬编码在代码里。建一个 `.env` 文件：

### 第三步：基本调用

用Claude 3.5 Sonnet，做一个最极简的非流式对话：

javascript
async function main() {
  const completion = await client.chat.completions.create({
    model: 'claude-3-5-sonnet-20241022',
    messages: [

  console.log(completion.choices[0].message.content);

main();


运行 `node app.js`，如果看到输出“太阳从东方升起”，恭喜你，第一步走通了。

### 第四步：流式输出（真正的生产力）

Claude的流式输出体验非常棒，它让你能逐字看到回答，而不是等它全部生完。这对做智能对话、实时工具来说至关重要。

javascript
async function streamMain() {
  const stream = await client.chat.completions.create({
    model: 'claude-3-5-sonnet-20241022',
    messages: [
      { role: 'user', content: '写一首关于程序员的短诗' }
    ],
    stream: true,
  });

  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
}

streamMain();


**防坑提醒**：流式输出模式下，返回的数据结构是 `{ choices: [{ delta: { content: '...' } }] }`。不要用 `chunk.choices[0].message.content`，那是在非流式模式下的用法。

### 第五步：处理文件/图片分析（Claude的多模态优势）

Claude 3.5 Sonnet 支持分析图片。把你本地的图片用Base64编码传进去就行。这个功能[千聚api聚合平台](https://www.qianjuai.com/)也完美支持。

javascript
const fs = require('fs');

async function imageMain() {
  const imageBase64 = fs.readFileSync('./test_image.png', { encoding: 'base64' });

  const completion = await client.chat.completions.create({
    model: 'claude-3-5-sonnet-20241022',
    messages: [
      {
        role: 'user',
        content: [
          { type: 'text', text: '请描述这张图片的内容。' },
          {
            type: 'image_url',
            image_url: {
              url: `data:image/png;base64,${imageBase64}`,
            },
          },
        ],
      },
    ],
  });

  console.log(completion.choices[0].message.content);
}

imageMain();


**防坑提醒**：图片大小不要太大。Claude官方限制单张图片不超过20MB，但为了速度，建议压缩在5MB以内。另外 `data:image/png` 这个格式前缀别忘了，否则图片无法被正确识别。

## 常见错误与解决方案（100%避坑指南）

### 错误1：HTTP 401 Unauthorized
- **原因**：API Key错误或未填写。
- **解决**：检查 `.env` 文件里的 `CLAUDE_API_KEY` 是否和[千聚api聚合平台](https://www.qianjuai.com/)控制台里的一模一样。前后有没有多余的引号或空格。

### 错误2：请求超时或卡住不动
- **原因**：国内网络波动，或者[千聚api聚合平台](https://www.qianjuai.com/)的节点响应慢。
- **解决**：首先确认你是不是在正常的国内网络环境下。然后可以设置请求超时时间：

javascript
const client = new OpenAI({
  apiKey: process.env.CLAUDE_API_KEY,
  baseURL: process.env.BASE_URL,
  timeout: 30000, // 30秒超时
  maxRetries: 3,  // 自动重试3次
});


### 错误3：返回内容被截断
- **原因**：超出了 `max_tokens` 限制。
- **解决**：显式设置 `max_tokens` 参数：

javascript
const completion = await client.chat.completions.create({
  model: 'claude-3-5-sonnet-20241022',
  messages: [/* ... */],
  max_tokens: 4096,  // 根据需求调整
});


### 错误4：提示“余额不足”或“配额耗尽”
