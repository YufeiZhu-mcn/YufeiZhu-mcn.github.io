## 第五步：最佳实践与异常处理——官方推荐写法

一个稳定的生产环境应用，不能没有健壮的异常处理。千聚的中转API同样遵循了OpenAI的标准错误码体系，因此你可以编写精准的容错代码。

**官方推荐的完整异常处理逻辑：**

python
import time
from openai import OpenAI, APIError, RateLimitError, APITimeoutError

client = OpenAI(
    api_key="YOUR_QIANJU_KEY",
    base_url="https://www.qianjuai.com/v1"
)

def safe_query(prompt, max_retries=3):
    """
    安全查询函数：涵盖超时、速率限制、通用API错误和网络问题
    """
    for attempt in range(max_retries):
        try:
            response = client.chat.completions.create(
                model="deepseek-r1",
                messages=[{"role": "user", "content": prompt}],
                timeout=20  # 设置超时时间，避免长时间挂起
            )
            return response.choices[0].message.content
            
        except APITimeoutError:
            print(f"请求超时（第{attempt + 1}次尝试）...")
            if attempt < max_retries - 1:
                time.sleep(2 ** attempt)  # 指数退避：等2秒,4秒,8秒...
            else:
                return "错误：请求超时，请检查网络或稍后重试。"
                
        except RateLimitError:
            print(f"触发速率限制（第{attempt + 1}次尝试）...")
            wait_time = 4 ** attempt
            time.sleep(wait_time)
            
        except APIError as e:
            print(f"API返回错误：{e.status_code} - {e.message}")
            # 如果是401错误，检查API Key是否有效
            if e.status_code == 401:
                return "错误：API Key 无效或已过期，请检查千聚控制台。"
            elif e.status_code == 429:
                # 429 通常是余额不足或并发超限
                return "错误：账户余额不足或并发请求超限。"
            else:
                return f"API错误：{e.message}"
                
        except ConnectionError:
            print("网络连接中断，检查能否正常访问 www.qianjuai.com...")
            time.sleep(5)
            
    return "错误：多次重试后仍然失败。"

**安全建议：**
- **API Key 绝对不要硬编码在代码里**。使用环境变量（`os.getenv()`）或密钥管理服务。
