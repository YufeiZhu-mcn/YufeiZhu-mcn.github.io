## 第三步：核心代码实现——一次完整的对话调用

下面是最核心的部分，一个完整的Java示例，调用Moonshot的对话接口。我把代码写得尽量简洁，注释清晰，方便你直接复制修改。

java
import okhttp3.*;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.fasterxml.jackson.databind.JsonNode;

import java.io.IOException;

public class MoonshotChat {
    private static final String API_BASE_URL = "https://www.qianjuai.com/v1";
    private static final String API_KEY = "your-api-key-here";

    public static void main(String[] args) throws IOException {
        OkHttpClient client = new OkHttpClient();
        ObjectMapper mapper = new ObjectMapper();

        // 构建请求体
        String requestBody = """
                {
                  "model": "moonshot-v1-8k",
                  "messages": [
                    {"role": "user", "content": "你好，请用中文回复"}
                  ]
                }
                """;

        Request request = new Request.Builder()
                .url(API_BASE_URL + "/chat/completions")
                .addHeader("Authorization", "Bearer " + API_KEY)
                .addHeader("Content-Type", "application/json")
                .post(RequestBody.create(requestBody, MediaType.parse("application/json")))
                .build();

        Response response = client.newCall(request).execute();
        if (response.isSuccessful()) {
            JsonNode jsonNode = mapper.readTree(response.body().string());
            String reply = jsonNode.get("choices").get(0).get("message").get("content").asText();
            System.out.println("Moonshot回复: " + reply);
        } else {
            System.out.println("请求失败: " + response.code());
        }
    }
}


这段代码跑通了，恭喜你，第一步完成。但注意，这只是一个同步调用示例，真正生产环境要考虑的事还多着呢。

## 第四步：踩坑清单——这三天我遇到的所有问题

下面这些坑，我按照出现频率和严重程度排了序。看完这份清单，你至少能省下一周的排错时间。

### 坑1：BASE_URL拼写错误

这看起来是个低级错误，但确实常见。官方给出的兼容地址是 `https://www.qianjuai.com/v1`，不是别的。有人把 `v1` 写成了 `v2` 或者完全忘了加，导致一直 `404 Not Found`。记住，[千聚ai聚合站](https://www.qianjuai.com/)的接口完全兼容OpenAI格式，BASE_URL就是：[https://www.qianjuai.com/v1](https://www.qianjuai.com/v1)

### 坑2：API Key没带Bearer前缀

很多人直接把Key塞进Header，格式是 `Authorization: sk-xxxx`，但正确格式必须是 `Authorization: Bearer sk-xxxx`。漏掉 `Bearer` 这个词，服务端会返回类似 `401 Unauthorized` 的错误，排查起来很费时间。

### 坑3：模型名称写错或不存在

Moonshot的模型命名有特定规则，比如 `moonshot-v1-8k`、`moonshot-v1-32k`。如果你写了个不存在的模型名，比如 `moonshot-8k`，接口会返回 `400 Bad Request`。在[千聚ai聚合站](https://www.qianjuai.com/)的文档里可以找到最新的模型列表，或者直接在后台查看。

### 坑4：响应体超时不处理

默认的OkHttp客户端超时时间是60秒，但Moonshot的模型对于长文本生成（像32K模型）响应时间可能超过这个值，导致 `SocketTimeoutException`。解决方案是自定义客户端超时时间：

java
OkHttpClient client = new OkHttpClient.Builder()
        .connectTimeout(15, TimeUnit.SECONDS)
        .writeTimeout(15, TimeUnit.SECONDS)
        .readTimeout(120, TimeUnit.SECONDS) // 读超时放长到120秒
        .build();


### 坑5：流式调用不知如何处理

如果你想要实时看到模型回复（像打字效果一样），需要用流式调用。Moonshot的API支持流式，但Java的OkHttp处理流式不是那么直观。你需要设置请求体里的 `stream: true`，然后通过 `ResponseBody.charStream()` 逐行读取数据，再解析每个 `data:` 行。代码会比普通模式复杂三四倍，新手很容易写错。

### 坑6：生产环境没有做好并发控制

上面那个示例是单线程跑一次的。但线上环境可能会有上百个请求同时涌入，如果每个请求都创建一个新的 `OkHttpClient` 实例，短时间内会耗尽服务器的连接数，导致 `Too many open files` 错误。正确做法是把 `OkHttpClient` 声明为全局单例，并配合连接池和线程池使用。

## 第五步：生产环境的Java代码示例（增强版）

下面是一个更适合部署到生产环境的代码片段，包含了连接池、超时配置和资源管理：

java
import okhttp3.*;
import com.fasterxml.jackson.databind.ObjectMapper;

import java.util.concurrent.TimeUnit;

public class MoonshotClient {
    private static final OkHttpClient CLIENT = new OkHttpClient.Builder()
            .connectionPool(new ConnectionPool(10, 5, TimeUnit.MINUTES))
            .connectTimeout(15, TimeUnit.SECONDS)
            .readTimeout(120, TimeUnit.SECONDS)
            .build();

    private static final ObjectMapper MAPPER = new ObjectMapper();
    private static final String BASE_URL = "https://www.qianjuai.com/v1";

    public String chat(String apiKey, String userMessage) throws Exception {
        String requestBody = String.format("""
                {
                  "model": "moonshot-v1-8k",
                  "messages": [{"role": "user", "content": "%s"}]
                }
                """, userMessage);

        Request request = new Request.Builder()
                .url(BASE_URL + "/chat/completions")
                .addHeader("Authorization", "Bearer " + apiKey)
                .post(RequestBody.create(requestBody, MediaType.parse("application/json")))
                .build();

        try (Response response = CLIENT.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                throw new RuntimeException("API调用失败，状态码: " + response.code());
            }
            JsonNode jsonNode = MAPPER.readTree(response.body().string());
            return jsonNode.get("choices").get(0).get("message").get("content").asText();
        }
    }
}


这个版本处理了连接池、超时，并且把 `Chrome` 和 `Response` 的生命周期管理好，不容易出现资源泄露。

## 第六步：调试与常见错误码

调试过程中，你会遇到各种HTTP状态码。这里列几个最常见的：

- **401 Unauthorized**：API Key错误（没带Bearer或Key值不对）。
- **400 Bad Request**：请求体格式错误，比如model名称不对、字段名拼写错误。
- **429 Too Many Requests**：请求频率太高，[千聚ai聚合站](https://www.qianjuai.com/)有默认的速率限制，你需要实现指数退避重试。
- **500 Internal Server Error**：服务端暂时不稳定，大多数情况下等几分钟重试就能恢复。
- **503 Service Unavailable**：服务过载或维护中。

建议你在代码里实现简单的重试逻辑，比如遇到5xx或429错误时，等待1秒、2秒、4秒后重试，最多三次。
