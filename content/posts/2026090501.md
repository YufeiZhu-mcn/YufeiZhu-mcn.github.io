### 坑一：推荐机制被“收税”

很多大型 API 聚合平台，本质上是个“市场”，入驻的服务商可以自由定价，平台抽水。这就导致一个结果：谁出价高，谁排在搜索结果前面。你看到的所谓“热门推荐”，很可能是最贵的那个。搜索 Qwen3 API key 购买，排名靠前的链接未必是性价比最高的。

### 坑二：所谓的“稳定节点”变成“静态额度”

一些平台会给你一个看上去很低的报价，但仔细一看，写着“静态额度”或“资源包”。意思是，你先买一个固定的 Token 包，用完即止，不能灵活充值。这种模式下，你一次性付清了钱，服务商提前拿到了现金流，他们当然乐意给你便宜点。但问题是，如果你项目上线后流量突然增长，你还得再单独购买新包，来回折腾不说，综合成本绝对不低。

### 坑三：渠道混乱，有逆向、有官转、有直连

Qwen3 的 API 其实有几种获取方式：从阿里云官方直连、从华为云 Azure 渠道调用、或者从其他云服务商中转。每个渠道的定价、稳定性、延迟都不一样。有些平台把“逆向”渠道（非官方授权的变通方法）和“官转”渠道（官方授权的重定向）混在一起卖，你根本不知道自己买到的是什么。逆向渠道虽然便宜，但随时可能被封或者被限流，说断就断。

### 坑四：隐藏扣量

这是我见过最恶心的一种操作。API 调用的计费依据是 Token 数，但一些平台会“故意多算”你消耗的 Token。比如你实际用了 1000 个 Token，平台后台记录成 1200 个。这种扣量差额积累下来，一个月白白多花 30%-50% 的钱。而 Qwen3 这类开源模型，由于本身价格就不高，扣量后单价直接就翻倍了。
