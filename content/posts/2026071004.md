## 第三步：配置API连接，只需改一行代码

这是最关键的一步，也是整篇文章最“爽”的地方。你不需要学习任何复杂的配置，只需要在你初始化OpenAI Service对象时，把`baseUrl`改成千聚的接口地址。

原来你可能是这样写的：

java
OpenAiService service = new OpenAiService("你的API_KEY");


现在我们改成这样：

java
import com.theokanning.openai.service.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.time.Duration;
import java.util.List;

public class Gpt4oJavaDemo {
    public static void main(String[] args) {
        // 1. 使用千聚中转站的API地址，并设置超时时间
        String apiKey = "你的千聚API_KEY"; // 🔴 替换成你复制的Key
        String baseUrl = "https://www.qianjuai.com/v1"; // 🔴 关键：换这里！
        
        // 2. 构建OpenAiService对象，传入自定义URL
        OpenAiService service = new OpenAiService(baseUrl, apiKey, Duration.ofSeconds(30));

        // 3. 构建聊天请求
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("gpt-4o") // 指定模型为gpt-4o
                .messages(List.of(
                        new ChatMessage("system", "你是一个Java编程导师，回答问题简洁明了。"),
                        new ChatMessage("user", "用Java写一个斐波那契数列方法，要求使用递归。")
                ))
                .maxTokens(500)
                .build();

        // 4. 调用并打印结果
        service.createChatCompletion(request).getChoices().forEach(choice -> {
            System.out.println("\n--- GPT-4o 回答开始 ---");
            System.out.println(choice.getMessage().getContent());
            System.out.println("--- GPT-4o 回答结束 ---\n");
        });
    }
}

## 高级用法：流式输出和错误处理

对于Java项目，流式（Streaming）输出能带来更好的用户体验。你只需要在请求构建时开启`stream`属性：

java
ChatCompletionRequest request = ChatCompletionRequest.builder()
        .model("gpt-4o")
        .stream(true) // 🔴 开启流式输出
        .messages(List.of(/* ... */))
        .build();

// 在回调中处理每一块流数据
service.streamChatCompletion(request)
    .doOnError(Throwable::printStackTrace)
    .blockingForEach(chunk -> {
        chunk.getChoices().forEach(choice -> {
            // 打印增量内容
            System.out.print(choice.getMessage().getContent());
        });
    });


同时，建议在代码里加入适当的异常捕获和处理逻辑，比如重试机制，让你的生产环境接入更健壮。
