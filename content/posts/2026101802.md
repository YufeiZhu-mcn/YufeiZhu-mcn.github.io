## Java调用示例：真的只是改几行代码

很多Java开发者最怕的就是环境配置。千聚最让我舒服的一点是：**完全兼容OpenAI接口**。所以经典的Java HTTP调用代码直接搬过来用，基本不费劲。

下面是一个基于`HttpClient` + `Jackson`的Java示例，调用GLM-4-9B模型，实现“一句话总结文本”：

java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import com.fasterxml.jackson.databind.ObjectMapper;
import java.util.Map;
import java.util.List;

public class GlmApiExample {
    public static void main(String[] args) throws Exception {
        // 1. 替换你的API Key
        String apiKey = "你的千聚API Key";
        // 2. 替换base_url
        String baseUrl = "https://www.qianjuai.com/v1";

        // 构建请求体
        ObjectMapper mapper = new ObjectMapper();
        Map<String, Object> body = Map.of(
            "model", "glm-4-9b-chat",
            "messages", List.of(
                Map.of("role", "user", "content", "简单总结一下：Java是一种广泛使用的编程语言，通过千聚API可以低至0.003元/次调用GLM模型。")
            ),
            "max_tokens", 100
        );

        String jsonBody = mapper.writeValueAsString(body);
        
        // 发送请求
        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(baseUrl + "/chat/completions"))
            .header("Content-Type", "application/json")
            .header("Authorization", "Bearer " + apiKey)
            .POST(HttpRequest.BodyPublishers.ofString(jsonBody))
            .build();

        HttpResponse<String> response = client.send(request, 
            HttpResponse.BodyHandlers.ofString());

        System.out.println("响应状态码: " + response.statusCode());
        System.out.println("响应数据: " + response.body());
    }
}

**接入关键点**：
- 把`base_url`从OpenAI的官方地址，换成`https://www.qianjuai.com/v1`。
- 把API Key换成千聚生成的key。
- 模型名改成`glm-4-9b-chat`。
- 运行代码。您的微服务、Spring Boot项目就可以一键调用GLM了。

如果你用Spring Cloud或FeignClient，直接用OpenAI的配置，修改`url`和`apiKey`属性，同样无缝衔接。
