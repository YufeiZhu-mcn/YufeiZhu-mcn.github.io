## 3分钟跑通Demo：Java接入o3示例全流程

下面就是核心代码环节。先声明一下：[千聚ai聚合站](https://www.qianjuai.com/)的API接口完全兼容OpenAI标准，所以我直接用了OpenAI官方的Java SDK，只改了配置参数。你不需要额外引入什么魔改过的中间件。

我先打开pom.xml，加上核心依赖：

xml
<dependency>
    <groupId>com.github.rectangle-db</groupId>
    <artifactId>openai-java</artifactId>
    <version>1.1.1</version>
</dependency>


这是一个轻量级的Java库，封装了OpenAI的HTTP调用，支持流式非流式，我用了它。

然后写主类代码。我起名叫 `O3Demo.java`，内容不长，但每一行都很关键：

java
import io.github.rectangledb.openai.client.OpenAiClient;
import io.github.rectangledb.openai.client.OpenAiConfig;
import io.github.rectangledb.openai.domain.chat.ChatCompletionRequest;
import io.github.rectangledb.openai.domain.chat.ChatCompletionResult;
import io.github.rectangledb.openai.domain.chat.Message;

public class O3Demo {
    public static void main(String[] args) {
        // 1. 配置OpenAI客户端，指向[千聚ai聚合站](https://www.qianjuai.com/)的API地址
        OpenAiConfig config = new OpenAiConfig();
        config.setApiKey("sk-你的千聚APIKey");  // 替换为你自己申请的Key
        config.setApiHost("https://www.qianjuai.com/v1");  // 关键：指向[千聚ai聚合站](https://www.qianjuai.com/)

        OpenAiClient client = new OpenAiClient(config);

        // 2. 构建请求，指定使用o3模型
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("o3")   // 模型名称：o3
                .message(Message.builder()
                        .role("user")
                        .content("Hello, 请用100以内素数举一个Java代码示例。")
                        .build())
                .maxTokens(200)   // 限制输出长度，避免消耗过多额度
                .temperature(0.7)
                .build();

        // 3. 发送请求，获取回复
        ChatCompletionResult result = client.chatCompletion(request);

        // 4. 输出结果
        System.out.println("o3 回复内容：");
        System.out.println(result.getChoices().get(0).getMessage().getContent());
    }
}


光看着代码就觉得清爽吧？没有代理配置、没有环境变量、没有脚本，就是最简单直接的HTTP调用。你用OpenAI官方SDK写出来的代码，跟这个唯一的区别就是 `apiHost` 指向的地方变了。

保存好代码后，把API Key换成你从[千聚ai聚合站](https://www.qianjuai.com/)后台复制过来的那个字符串。这是唯一需要你手动修改的参数。

跑一下这个 `main` 方法。控制台里很快就打印出来了——o3成功响应，返回了一段Java代码示例，并且附带了解释说明。全过程中没有报超时异常、没有网络连接失败、没有证书错误。干干净净的。

## 常见问题排查

我第一次跑的时候也遇到一个乌龙——API Key填错了。后台复制Key的时候多复制了一个空格，结果一直报401 Unauthorized。处理方式很简单：删除Key首尾的空格,重新粘贴一次。这是最坑的常见问题，没有之一。

还有同学可能会遇到 **连接超时** 或 **DNS解析失败** 的问题，这个大概率是本地网络防火墙或企业网络策略拦截了海外流量。我的解决办法是：用4G热点试一下，如果热点能通，说明是企业内网限制；如果热点也不行，检查一下系统HOSTS或防火墙设置。

另外，[千聚ai聚合站](https://www.qianjuai.com/)的API接口需要HTTPS访问，如果你的Java环境证书有问题，可以用简单的方式调一下JVM参数跳过证书校验（不推荐，只做快速验证的话可以试，线上环境必须配好证书）。
