## 第一种方式：使用 Anthropic 官方 Java SDK（最正统，但门槛高）

Anthropic 官方确实提供了 Java SDK，也就是 `anthropic-java`。如果你是海外环境，有海外的服务器和信用卡，直接用官方 SDK 当然是最正统的做法。

**接入示例（伪代码）**：
java
// 引入官方依赖
// 需要配置 ANTHROPIC_API_KEY 环境变量
ClaudeClient client = ClaudeClient.builder()
    .apiKey("sk-ant-xxxxx")
    .build();

// 调用 Claude Opus 4.1
Message message = client.messages()
    .model("claude-opus-4.1-20250224")
    .content("你好，请用Java写一个二分查找")
    .execute();


**优点**：
- 100% 官方最新功能，新模型第一个支持。
- 有官方文档和技术支持。

**缺点**：
- 国内直接没法用，必须配置科学上网或者海外服务器。
- 价格最贵，官方原始定价，没有任何折扣。
- 开通账号、绑定信用卡、避免封号，每一步都是坑。

**适合人群**：有钱有海外资源的纯技术爱好者，或者真的只差官方原生功能的特殊场景。

## 第三种方式：通过 AWS Bedrock 接入（企业级，但需要 AWS 账号）

如果你本身就在用 AWS 全家桶，通过 Amazon Bedrock 调用 Claude Opus 4.1 也是个办法。AWS 的 Bedrock 服务原生集成了 Anthropic 的模型。

**接入示例**：
java
// 需要引入 AWS SDK for Java v2
// 配置 BedrockRuntimeClient

BedrockRuntimeClient client = BedrockRuntimeClient.builder()
    .region(Region.US_EAST_1)
    .credentialsProvider(DefaultCredentialsProvider.create())
    .build();

// 调用 Claude
String response = client.invokeModel(invokeModelRequest).body();


**优点**：
- 企业级稳定性，有 AWS 背书。
- 数据合规，适合有合规要求的公司。

**缺点**：
- 国内要用 Bedrock，一样需要跨国网络配置。
- 开通复杂，AWS 的 IAM 权限配置就够折腾半天。
- 成本不低，AWS 的 API 调用还有额外通道费用。

**适合人群**：已经在用 AWS、并且公司有合规数据落地要求的团队。个人开发者不推荐。

## 第五种方式：HTTP 直接调用（最原始，最灵活）

不想引入任何第三方 SDK？没问题，直接发 HTTP 请求给 Anthropic 官方接口，或者给中转平台的 OpenAI 兼容接口。

**用 Java HttpClient 示例**：
java
HttpClient client = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://www.qianjuai.com/v1/chat/completions"))
    .header("Content-Type", "application/json")
    .header("Authorization", "Bearer 你的Key")
    .POST(HttpRequest.BodyPublishers.ofString(jsonBody))
    .build();


**优点**：
- 没有额外的 SDK 依赖，项目干净。
- 完全可控，可以自己实现流式、重试、降级逻辑。

**缺点**：
- 代码量大，所有东西都要自己手写。
- 没有类型安全，JSON 序列化反序列化容易出错。
- 没有官方的流式支持，需要自己解析 SSE。

**适合人群**：想学习底层原理，或者项目对依赖管理有严格限制的极客。生产环境不推荐。
